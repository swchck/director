# Security Policy

## Supported Versions

| Version | Supported |
|---|---|
| Latest on `main` | Yes |

## Reporting a Vulnerability

If you discover a security vulnerability in Director, please report it responsibly.

**Do NOT open a public GitHub issue for security vulnerabilities.**

Instead, please use [GitHub Security Advisories](https://github.com/swchck/director/security/advisories/new) to report the vulnerability privately.

### What to include

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

### Response timeline

- **Acknowledgment**: within 48 hours
- **Initial assessment**: within 1 week
- **Fix and disclosure**: coordinated with reporter

## Scope

This policy covers the `director` Go library and its sub-packages. It does not cover:

- Third-party dependencies (report to their maintainers)
- Directus CMS itself (report to the [Directus team](https://directus.io))
- Example code (provided for illustration, not production use)
